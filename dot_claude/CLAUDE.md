# 作業ガイドライン

## bashコマンド

bashコマンドはなるべく単体（&&でつなげずに）で実行する。こうすることで許可が容易になる。

## Git

- ブランチ名はasciiキャラクターのみを使うこと。日本語を使うと各種コマンドが失敗することがある
- フィーチャーブランチはfeature/<branch-name>形式を使う
- 対応するタスクが存在する場合コミットメッセージは MAI-12: のようにタスクIDから始める
- gitコマンドはサンドボックス内で実行できる。サンドボックス外では認証に失敗するので注意。
gitコマンドが認証エラー（401, 403, "Bad credentials"等）で失敗した場合は、GitHub Appトークンの期限切れの可能性がある。
`just -g claude-update-gh-token`で更新できる。

## gh
- ghコマンドはサンドボックス外で実行する必要がある
- 承認なしで実行するため、以下のルールを守ること:
  - コマンドは1つずつ実行する（`&&` や `;` で繋げない）
  - 変数展開を含めない（`$TMPDIR` 等は使わずリテラルパスを使う）
  - 複数行テキスト（PR本文等）はファイル経由で渡す（`--body-file`, `--input` 等）
  - ファイル経由のフラグがないサブコマンドでは `--body` に直接渡してよい（#行を含む場合は承認が発生しうる）
